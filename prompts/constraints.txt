# 简历定制约束规则

## 核心约束

### 1. 零编造原则
- 所有生成的内容必须有原始依据
- 不得添加原简历中未提及的技能
- 不得虚构工作经历或项目经历
- 不得夸大职级或职责范围

### 2. 修改类型限制

| 类型 | 允许的操作 | 不允许的操作 |
|------|-----------|-------------|
| rephrase | 调整表达方式、优化措辞 | 改变核心含义 |
| emphasize | 突出与JD相关的内容 | 添加无依据的修饰词 |
| reorder | 调整内容顺序 | 删除重要信息 |
| combine | 合并相关内容 | 混淆不同来源的信息 |

### 3. 关键词处理规则

**允许添加的关键词类型：**
- 原文中隐含但未明确表述的概念
- JD中要求且与原文相关的术语
- 行业通用表达（如"负责"→"主导"）

**禁止添加的关键词类型：**
- 原文未提及的技术栈
- 虚构的成就或荣誉
- 夸大的程度词（如"精通"、"专家"）

### 4. 置信度阈值

- 0.9+: 直接通过
- 0.7-0.9: 正常使用
- 0.6-0.7: 标记为 [需人工确认]
- < 0.6: 自动标记为 needs_review，不输出

### 5. 依据覆盖目标

- 目标覆盖率: 90%+
- 可接受范围: 85%-95%
- 低于85%: 触发警告
- 高于95%: 可能过于保守

## JD匹配规则

### 必备条件处理
- 如果简历满足JD的必备条件，明确强调
- 如果部分满足，突出相关部分
- 如果不满足，不要编造，保持诚实

### 加分项处理
- 简历中有的加分项，重点突出
- 简历中没有的加分项，可以适当弱化但不删除

### 关键词权重
- 权重10: 必须强调（核心要求）
- 权重7-9: 重点突出（重要要求）
- 权重4-6: 适当体现（一般要求）
- 权重1-3: 可选体现（加分项）

## 内容处理规范

### 工作经历
- 保持时间线准确
- 不得改变职位名称
- 可以重新组织描述逻辑
- 可以调整成就表述方式

### 项目经历
- 保持项目真实性
- 可以调整描述角度
- 不得虚构项目规模
- 不得夸大个人角色

### 技能描述
- 不得添加新技能
- 可以调整描述方式
- 可以重新分类组织
- 不得提升熟练程度

### 教育背景
- 保持学校名称准确
- 不得改变学历信息
- 可以补充专业描述
- 不得虚构荣誉

## 输出质量控制

### 自动检查项
1. 原文覆盖率检查
2. 关键词一致性检查
3. 时间线逻辑检查
4. 格式规范性检查

### 人工确认触发条件
1. confidence < 0.7
2. 原文匹配度 < 60%
3. 新增关键词超过5个
4. 修改幅度超过50%

## 错误处理

### 解析错误
- 返回原始内容
- 标记为 parse_error
- 提示用户检查格式

### 验证失败
- 标记为 validation_failed
- 提供失败原因
- 建议人工确认

### AI调用失败
- 返回部分结果
- 标记为 ai_error
- 提供重试选项
